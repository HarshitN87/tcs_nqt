Hiring Tiers & Packages

TCS hires into three distinct packages based on NQT performance. No negative marking — fill every response.

Prime
₹9 – ₹12 LPA

Flawless advanced aptitude (>80%) + both coding questions fully solved (7/7 test cases each). Zero margin for error — a single failed hidden test case drops you to Digital.

Digital
₹7 – ₹7.5 LPA

Advanced aptitude >70-75% + at least 1 coding question fully solved. Passing all hidden cases on the 50-mark algorithmic question alone can get you here even if you fail Q1 entirely.

Ninja
₹3.36 – ₹3.6 LPA

Cleared basic cognitive cutoffs (~60%) + partial or empty coding responses. Fractional test case completion (5/7 or 6/7 on both questions) almost universally routes here.

Platform Constraints & Compiler Quirks
The Blank Slate Notepad. The coding window is a completely blank canvas — zero autocomplete, no syntax highlighting, no linting, no auto-closing brackets. You must manually select your language from a dropdown and activate the console setup.
  • Execution Lag: During peak exam windows, the compiler suffers severe latency. The terminal can freeze or display a generic "executing" status, completely withholding test-case results until it's too late to debug.
  • No Version Checkpoints: No backup or undo history. If your solution passes 3/7 test cases and you attempt a risky last-minute optimization, any error introduced right before the timer ends cannot be reverted — leaving you with zero marks for an otherwise salvageable solution.
  • On-Screen Calculator: A basic system calculator is provided for aptitude, but manipulating it via mouse clicks slows down operations significantly.
The Input-Parsing Trap
This causes more failures than the actual core logic. Unlike LeetCode or campus tools that give you a pre-written driver function, TCS requires building the entire program from scratch — header files, main function, and raw stdin parsing.
  • Unstructured Arrays: The test engine frequently omits the initial integer specifying array size (N), passing only space-separated or comma-separated integers. Standard fixed loops crash if N is not provided.
  • The Workaround: In C++, use while(cin >> x) or getline + stringstream to dynamically populate a vector. In Python, .split() and list comprehensions handle volatile input naturally.
  • Language Preference: Reddit users strongly advocate using Python for the OA — parsing volatile inputs with .split() reduces boilerplate and minimizes errors under time pressure, provided you practice writing it without IDE assistance.
Output Formatting & Edge-Case Traps

Public test cases carry zero score weight — your evaluation relies entirely on hidden test sets.

  • Decimal Formatting: Codes that compute correct math fail because the engine expects values rounded to exactly 2 decimal places — or trailing zeros stripped completely (e.g., 11.00 must print as "11", and 10.50 as "10.5").
  • String Literal Flags: Problems hide a conditional exception at the very bottom. If evaluation results in negative/invalid range, you must return exact strings like "Error" or "Invalid range". Missing this single if block guarantees failure on majority of hidden cases.
  • Arbitrary Boundaries: In range-filtering problems, hidden tests may require single-digit integers (numbers < 10) to be skipped entirely based on an unstated backend rule.
Scoring, Routing & The Partial Score Trap
Prime
Coding: 2/2 questions fully solved, 100% hidden cases on both
Aptitude: Advanced >80%, Standard high accuracy
Digital
Coding: 1/2 fully solved + partials on the other
Aptitude: Advanced >70-75%, Standard positive baseline
Ninja
Coding: 0-1 questions partially solved
Aptitude: Standard cognitive >60% minimum
The Easy/Hard Paradox. Many candidates pass 7/7 on the complex 50-mark question but score 0/7 on the basic 25-mark question due to input formatting bugs. A flat zero on the implementation problem flags you as a "foundational failure" — systematically stripping Prime eligibility and routing you to Ninja regardless of algorithmic performance.
  • The "One-Question" Digital Path: If you completely fail Q1 but ace Q2 (50 marks, 100% test cases), the system automatically shortlists you for Digital, bypassing the Q1 failure.
  • Partial Score Penalty: Passing 6/7 on both questions mathematically scores higher than solving just one, but the routing logic penalizes incomplete execution — fractional completion on both problems drops you to Ninja, because the filter looks for absolute success on high-value problems.
  • Aptitude as Gatekeeper: Even with flawless 2/2 coding, if your advanced aptitude falls below ~80%, the system demotes you from Prime to Digital retroactively.
